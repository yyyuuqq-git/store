======================================================================
펫플래닛 (Pet Planet) 개발에 사용된 핵심 프롬프트 컬렉션 (Prompts Collection)
======================================================================

본 문서에는 펫플래닛 프로젝트의 기획부터 기초 설계(블루프린트), 기능 구현, 
그리고 최종 어드민 기능 추가까지 개발 과정 전반에 사용된 핵심 프롬프트들이 정리되어 있습니다.
발표 시 시연 및 설명 자료로 활용하시기 바랍니다.

----------------------------------------------------------------------
1. 기초 설계 단계: 프롬프트 블루프린터 (Meta-Prompt)
----------------------------------------------------------------------
[설명] 
개발 시작 전, LLM이 최적의 웹 어시스턴트로 작동할 수 있도록 역할을 정의하고 
전체적인 디자인 시스템과 아키텍처 설계도를 먼저 생산하도록 지시한 메타 프롬프트입니다.

[프롬프트 본문]
"너는 지금부터 시니어 웹 개발자이자 소프트웨어 아키텍트이다. 우리는 반려동물 케어&쇼핑 플랫폼인 '펫플래닛'을 HTML, Vanilla CSS, Vanilla JS를 활용해 제작할 것이다. 
코드를 바로 작성하지 말고, 먼저 다음 요구사항에 맞춰 '개발 블루프린트(설계도)'를 작성해다오.
1. HSL 색상 체계를 사용한 다크/라이트 테마 프리미엄 디자인 시스템 정의 (Glassmorphism 스타일 포함)
2. 전체 애플리케이션의 파일 구조 설계 (index.html, app.js, style.css, supabase-client.js)
3. 회원 관리를 위한 Supabase DB 테이블 정의 (profiles) 및 트리거 로직 설계
4. 이중 카테고리(강아지/고양이 -> 세부 용품) 필터링 인터페이스 설계
이 도면이 완성되면 사용자 컨펌을 거쳐 단계별로 코드를 구현할 것이다."

----------------------------------------------------------------------
2. 데이터베이스 연동 단계: Supabase Auth 및 사용자 정보 연동
----------------------------------------------------------------------
[설명]
회원가입 시 단순히 이메일뿐만 아니라 보호자 이름, 연락처 정보를 메타데이터로 함께 넘겨 
안전하게 DB 프로필로 동기화되게 유도하는 클라이언트 구축 프롬프트입니다.

[프롬프트 본문]
"Supabase JS SDK를 활용하여 회원가입(Sign Up) 및 로그인(Sign In)을 처리하는 supabase-client.js 파일을 작성해 줘.
회원가입 시 사용자 메타데이터(raw_user_meta_data)에 '보호자 이름(name)'과 '연락처(phone)'를 담아서 가입시켜야 해. 
그리고 이 가입 정보가 Supabase Auth에 생성됨과 동시에 public.profiles 테이블로 복사되는 PostgreSQL 트리거 함수와 테이블 구조(SQL DDL)도 supabase_setup.sql 파일로 정리해 줘. 
보안을 위해 RLS(Row Level Security) 정책을 활성화하여 사용자가 본인의 프로필 정보만 수정 및 조회할 수 있게 설정해야 해."

----------------------------------------------------------------------
3. 핵심 기능 구현: 반려동물 건강 진단 및 특화간식 처방 플래너
----------------------------------------------------------------------
[설명]
사용자가 반려동물의 품종과 걱정되는 건강 질환을 입력하면, 맞춤형 영양 가이드와 
특화 간식을 추천해 주는 인터페이스 구현 프롬프트입니다.

[프롬프트 본문]
"index.html의 플래너 영역에 반려동물 종류(강아지/고양이), 세부 품종, 가장 염려되는 질환(슬개골 탈구, 눈물샘, 아토피, 비만 등)을 선택할 수 있는 폼을 만들어 줘.
사용자가 '진단 및 추천간식 설계하기' 버튼을 누르면:
1. 수의사 추천 권장 영양 가이드 텍스트와 급여 팁이 시각적인 진단서 폼으로 렌더링되게 해 줘.
2. 스토어의 상품 중 해당 질환 케어에 매칭되는 특화 간식(예: 관절 케어용 츄, 눈물 개선용 간식 등)을 추천 상품으로 강조해서 리스트에 띄워 줘.
3. 이 모든 과정이 매끄럽고 모던한 마이크로 인터랙션과 함께 처리되도록 app.js를 작성해 줘."

----------------------------------------------------------------------
4. 관리자 권한 확장 단계: 상품 품절 처리 및 삭제 기능 고도화
----------------------------------------------------------------------
[설명]
관리자가 스토어 내에서 실시간으로 상품을 품절 처리하거나 완전히 제거할 수 있도록 권한을 검증하고 관련 UI/UX와 백엔드 트랜잭션을 연결한 프롬프트입니다.

[프롬프트 본문]
"관리자 계정으로 로그인한 상태일 때, 각 상품 카드에 관리자 전용 제어 박스(품절처리/판매재개 버튼, 삭제 버튼)를 노출해 줘.
동작 요건은 다음과 같아:
1. '품절처리'를 클릭하면 데이터베이스의 is_sold_out 값을 true로 업데이트하고, 해당 카드의 이미지를 회색톤(Grayscale)으로 흐리게 처리하며 중앙에 'SOLD OUT' 스탬프가 나타나게 해 줘. 이때 '장바구니' 버튼은 '품절 🚫'로 변경되고 비활성화되어야 해.
2. 품절 상태인 상품도 '🔍 상세 정보' 클릭은 가능해야 하지만, 상세 페이지(product-detail.html)에 들어가면 상단에 '⚠️ 품절된 상품입니다' 경고 배너가 표시되고 상세 페이지 장바구니 버튼도 비활성화되어야 해.
3. '삭제' 버튼을 누르면 컨펌 창을 띄운 뒤 Supabase products 테이블에서 해당 상품을 삭제하고, 홈페이지의 상품 리스트에서도 실시간으로 제거되도록 연동해 줘.
4. 개발자가 직접 삭제 및 품절 테스트를 수행해 볼 수 있도록, 첫 화면 가장 앞에 노출되는 테스트용 더미 상품 하나를 productsData 배열에 추가해 줘."
